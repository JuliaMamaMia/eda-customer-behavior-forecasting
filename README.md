# Customer Behavior Analytics & Revenue Forecasting (EDA · Stats · Time Series)

**TL;DR:** полный цикл аналитики по транзакциям: предобработка → EDA → гипотезы → RFM‑сегментация → прогноз выручки (ETS/временные ряды).  
**Stack:** Python (pandas/numpy/matplotlib/seaborn) · statsmodels/scipy · Excel‑friendly output  

---

## 1) Business question
Как понять поведение клиентов и драйверы выручки:
- какие сегменты приносят основную выручку,
- чем отличаются группы клиентов,
- как прогнозировать выручку на основе временных рядов.

## 2) Data
Транзакционные данные розничного магазина (покупки, клиенты, суммы, категории и т.д.).  
> Датасет не храню в репозитории (чтобы не засорять Git): в ноутбуке есть шаги подготовки и проверки качества данных.

## 3) Approach
1. **Предобработка:** типы данных, пропуски, дубликаты, аномалии/выбросы.
2. **EDA:** динамика выручки, распределения, новые признаки.
3. **Статистика:** проверка гипотез (сравнение групп), интерпретация результатов.
4. **Сегментация:** RFM‑анализ и выводы по сегментам.
5. **Прогнозирование:** подготовка ряда, проверка стационарности, выбор модели, валидация, финальный прогноз (ETS).

## 5) Results & recommendations
- Сформирована структура проверки данных (качество, выбросы, дубликаты) и воспроизводимый пайплайн анализа.
- Получены выводы по клиентским сегментам (RFM) и различиям групп (гипотезы).
- Построен прогноз выручки на временных рядах с валидацией и диагностикой остатков.
